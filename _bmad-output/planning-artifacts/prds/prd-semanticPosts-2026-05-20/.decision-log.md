### 2026-05-20 (later) — Grilling pass applied; PRD revised
User invoked `mattpocock-skills:grill-with-docs` and walked the design tree with thinking-partner mode. The session produced `docs/CONTEXT.md` (domain glossary) and seven ADRs (`docs/adr/0001-..0007`). Key decisions that propagate into the PRD:

- **ADR-0001 — Indexable Text composition.** Title repeated 3× + manual excerpt + cleaned content, truncated. Propagated into Glossary §3 (Indexable Text) and FR-1.
- **ADR-0002 — Hash-diff regeneration triggers.** `save_post` computes Indexable Text hash; only changed posts get re-embedded. Hourly cron processes dirty queue. First-publish bypass for immediate freshness. Replaces previous "embedding queued on every save_post" model. Propagated into FR-1.
- **ADR-0003 — Postmeta storage as Derived Data.** All `_sp_*` postmeta is regenerable from posts; not backup-worthy. Restore-without-data triggers automatic background reindex. Replaces previous backup-size handling. Propagated into Glossary §3 (Derived Data).
- **ADR-0004 — Crawler-based indexing.** Similarity Graph maintained incrementally via candidate-set walks. O(K²) per save, independent of corpus size. Cold start uses brute-force batched/resumable; weekly verification pass detects drift. **Replaces FR-4 entirely** — the old "daily WP-Cron full refresh" model is gone. New FRs: FR-4 (crawler), FR-4a (Ranking Mode selection), FR-4b (cold start), FR-5 (manual reindex / verification).
- **ADR-0005 — No per-embedding model versioning.** Changing the embedding model wipes all data and triggers cold start. No mixed-model state support. Propagated into FR-9 settings UI behavior.
- **ADR-0006 — Three configurable Ranking Modes.** Most relevant (default, pure cosine), Fresh-first (recency-boosted), Diverse mix (MMR). Featured #1 is always highest cosine across all modes. New FR-4a. Settings field added to FR-9.
- **ADR-0007 — Rendering contract.** HTML structure, CSS classes, theme override path (`{theme}/semantic-posts/related-posts.php`), customization filters/actions, `data-sp-source` diagnostic attributes treated as a public contract (additive-only post-launch). Propagated into FR-8.

Additional decisions:
- **Recommendation List size:** range expanded from 3–6 to 3–10. Quality-bounded mode added as opt-in (default off — predictable list sizes). Propagated into FR-8 and FR-9.
- **Recommendation Source:** silent category fallback when semantic source is unavailable, `data-sp-source` exposes provenance to admins. Closes Open Question #1 (empty-state UX). Propagated into FR-6, FR-8, UJ-2, §3 glossary.
- **Multi-language defensive support:** Polylang/WPML detection filters candidate set to same-language. Multi-language moved out of Non-Goals/Non-Users (defensive support is in scope; full multilingual Pro features remain out). Propagated into FR-4, §2.3, §5, §6.
- **Configurable post types (multi-select):** UI lists all public post types; default `post`. Commerce types get advisory note, not block. Propagated into FR-9 and §2.3.
- **Simpler error handling:** uniform exponential backoff (3 tries), failed-post counter + "Retry failed" button. Categorized error handling (401 vs 429 vs 5xx) and daily cost cap were considered and rejected as overkill for v1. Propagated into FR-3.
